1. Компилятор - это программа, которая читает код и создаёт автономную исполняемую программу, которую процессор понимает напрямую. При запуске программы весь код компилируется целиком, а затем создаётся исполняемый файл и уже при повторном запуске программы компиляция не выполняется. 

(Код высокоуровневого языка) - (Компилятор) - (Исполняемый файл) 

(Исполняемый файл) - (Процессор) - (Результат программы)



2. Интерпретатор - это программа, которая напрямую выполняет код, без его предыдущей компиляции в исполняемый файл.

(Код высокоуровневого языка) - (Интерпретатор) - (Процессор) - (Результат программы)



3. Перед написанием первой программы надо знать как выполняется разработка программного обеспечения на языке С++


(Определить проблему)[Что я хочу сделать?] - (Разработать решение)[Как я хочу сделать?] - (Написать программу, используя решение) - (Скомпилировать программу) - (Соединить объектные файлы) - (Протестировать и отладить)


4. Линкинг {Компоновщик} - это процесс связывания всех объектных файлов, генерируемых компилятором, в единую исполняемую программу, которую вы затем сможете запустить/выполнить.


5. Рабочее пространство - это своеобразный контейнер, который может содержать один или несколько связанных проектов.


6. Конфигурация сборки - это набор настроек проекта, которые определяют принцип его построения. Она содержит в себя:
    - имени исполняемого файла;
    - имени директории исполняемого файла;
    - имени директорий, в который IDE будет искать другой код и файлы библиотек;
    - информации об отладке и параметрах оптимизации вашего проекта;
 
 Debug - предназначена для отладки вашей программы.
 Release - используется во время сборки программы для её дальнейшего выпуска.


7. Стэйтмент - это строка кода состоящая из нескольких операторов, которое сообщает компилятору, что ему нужно выполнить определённое задание.


8. Выражение - это математический объект, который создаётся для проведения вычисления и нахождения соответствующего результата. 


9. Функции - это последовательность стэйтментов. 


10. Библиотека - это набор скомпилированного кода, который был "упакован" для повторного использования в других программах.

11. Директива препроцессора - это команда которая выполняется до того, как начнётся компиляция кода.


12. Синтаксис - правила, которые регулируют построение предложений, называют синтаксисом.


13. Оператор - это 1 объект.


14. Комментарий - строка текста, которая вставляется в исходных код для объяснения того, что делает код.


15. Однострочные комментарии - это комментарии которые пишутся после символов //. Они пишутся в отдельных строках и всё, что находится после этих символов комментирования, - игнорируется компилятором


16. Многострочные комментарии - это комментарии, которые пишутся между символами /* */. Всё что находится между звёздочками, игнорируется компилятором. 


17. Закомментированный код - это конвертировать одну или несколько строк кода в комментарии. Таким образом, вы можете (временно) исключать часть кода из компиляции.
